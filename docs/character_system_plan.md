# 캐릭터 설정 시스템 스펙 및 구현 계획 정리

이 문서는 비주얼 노벨 디스코드 애드온의 **캐릭터 스탠딩 설정 및 표정 관리 시스템**에 대한 통합 스펙 및 구현 사항을 정리한 문서입니다. (웹 대시보드 및 데모 환경 공통 적용)

## 1. 이미지 규격 (해상도 권장사항)
- **Base (몸통) 이미지**: `800 x 2160` 픽셀
- **Face Grid (표정 템플릿) 이미지**: `2400 x 3240` 픽셀 (가로 3열 × 세로 3행)

## 2. 표정 템플릿 분할 (Face Grid Slicer)
- 사용자가 `2400 x 3240` 크기의 표정 템플릿 이미지를 업로드합니다.
- 클라이언트(브라우저)에서 `<canvas>`를 사용하여 이를 **9등분(3x3)** 합니다.
- 분할된 각 조각의 크기는 **`800 x 1080`** 픽셀이 됩니다.
- 잘라낸 9개의 파츠는 Base 이미지 위(지정된 Anchor 좌표)에 오버레이 되어 하나의 전신 스탠딩 이미지를 완성합니다.

## 3. 표정 태그 시스템 (9+1 감정)
디스코드 명령어 및 이모지 매핑, 그리고 웹 UI에서 사용하는 9개의 영문 감정 태그 규격입니다.

| 인덱스 | 영문 태그명 | 설명 (매핑 예시) |
| :---: | :--- | :--- |
| 1 | **Neutral** | 기본, 무표정 (😐, 😶) |
| 2 | **Happy** | 웃음, 기쁨 (😊, 😄, 😂, 😁) |
| 3 | **Sad** | 슬픔, 우울 (😢, 😭, 😥) |
| 4 | **Angry** | 분노, 화남 (😡, 🤬, 😠) |
| 5 | **Surprised**| 놀람, 경악 (😲, 😱, 😮, 🤯) |
| 6 | **Embarrassed**| 당황, 부끄러움 (😳, 🫣) |
| 7 | **Scared** | 두려움, 공포 (😨, 😰) |
| 8 | **Thinking** | 고민, 생각 (🤔, 🧐) |
| 9 | **Custom** | 특수, 커스텀 표정 (✨) |
| - | **Ignore** | (분할된 슬롯 중 사용하지 않을 파츠를 무시할 때 선택) |

## 4. 앵커(Anchor) 지정 시스템
- Base 이미지 위에서 **얼굴의 중심 좌표**를 지정하여, 분할된 표정 파츠가 위치할 곳을 정합니다.
- 마우스 클릭을 통해 직관적으로 비율 좌표(`X%`, `Y%`)를 획득할 수 있습니다.
- 또한, 숫자 입력창(Input Number)을 통해 소수점 단위의 미세한 수동 좌표 조정이 가능합니다.
- 설정된 앵커 위치는 빨간색 십자선 마커로 시각화됩니다.

## 5. 반영된 파일 및 모듈
- **데모 환경 (`demo/index.html`)**
  - 바닐라 JS 기반의 앵커 피커 및 표정 슬라이서 로직 구현.
  - 전역 `state.chars`를 통해 표정 상태(`Neutral` 등) 관리 및 실시간 캔버스 합성 프리뷰 지원.
- **웹 대시보드 환경 (`apps/web/components/character/...`)**
  - React 기반의 `AnchorPicker.tsx`, `FaceGridSlicer.tsx`, `CharacterEditor.tsx` 컴포넌트로 구현.
- **공통 패키지 및 봇 (`packages/shared/src/index.ts`, `apps/bot/...`)**
  - `EMOJI_EXPRESSION_MAP`을 통해 디스코드 채팅 내 이모지 입력 시 자동으로 영문 태그로 변환.
  - `/표정` 슬래시 커맨드 및 `!Happy` 등의 수동 명령어를 통해 표정 전환 지원.
